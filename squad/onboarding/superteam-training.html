Before the Workshop

Brief the Squad in 15 Minutes

Before the client meeting. Do this first.

1
Tell Superman who the client is and what problem they have
2
Confirm the tech stack (Azure? M365? Custom?)
3
Set the squad mode: Solo for quick demos, Squad for feature builds
4
Superman runs the checklist. Agents align. No one starts until it's done.
Example prompt
"Superman — I'm preparing for a workshop with [Client]. They need [X]. Stack is [Azure / M365 / custom]. Brief the team for a PoC-on-the-fly."
In the Workshop

Capturing the Brief in Real Time

Client names the challenge. You turn it into a Superman brief.

What you hear
"We need to automate our supplier onboarding process"
"Our data is siloed — we can't see the full client picture"
"Can we build something to classify these documents automatically?"
What you say to Superman
"Superman — supplier onboarding PoC on Azure. Bob, what do we need?"
"Superman — unified client view, graph-based. Bob, blueprint it."
"Superman — document classifier PoC. Azure AI. Brief Athanasios first, then Eric."
Superman handles the routing. You handle the client.

The Red Team

Kryptonite Breaks It First

So the client doesn't.

What Kryptonite challenges
Assumptions in the brief ("what if the client can't give you live data?")
Architecture risks ("this won't scale past 100 users")
Demo risks ("what breaks if the internet drops?")
Scope creep ("are you building what they asked or what you think they need?")
“
A plan that survives Kryptonite is a plan worth building.

Emergency Setup

Need to Set Up During a Workshop?

If you don't have Superteam ready before the session — here's the fastest path.

1
Open claude.ai → New Project → name it Superteam
2
Go to github.com/stevemarksava/superteam
3
Open squad/onboarding/paste-into-claude.md → copy the file
4
Paste into Project Instructions → add your name and client context at the bottom
5
Say: "Superman — I'm [name], FDE at Avanade. I have a client brief: [X]. Go."
Setup time
3–5 minutes
Then the squad is live.
