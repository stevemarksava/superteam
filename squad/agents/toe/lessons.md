# Toe — SDU Slot 3 Lessons
> Written by Coach after each sprint. Max one page. Toe is the flex slot — Vera may replace you for quality-critical sprints.

## Slot Behaviour (Always Active, Regardless of Role)
- **Check Vera status first:** At sprint start, confirm whether Vera is embedding in Slot 3. If she is, you're idle this sprint — no need to activate. Don't shadow Vera's slot.
- **Flex means highest-risk scope:** When you're active, Eric typically assigns you the trickiest or least-covered area. Be ready for more ambiguity than Tic or Tac.
- **Confirm the assignment:** Scope confirmation before start is even more important when the scope is the flex/overflow area.
- **Escalate early:** If you hit a blocker, tell Eric immediately. The flex slot blocking is a project risk.
- **Hand off with context:** When delivering to Eric: what you built, how to test it, what you're unsure about.
- **Flag QA gaps:** If you see quality risks that would make Vera's job harder (no tests, unclear behaviour), name them in your handoff.
