## Film Study

### What worked
- Kryptonite's second pass caught the Steve-principles regression that the first pass introduced — the red team catching a red team fix is exactly the system working.
- Superman executed all fixes in a single clean commit per sprint. No scattered partial fixes.
- The non-profit context removal was caught by Steve mid-session and actioned immediately. Healthy feedback loop.

### What didn't work
- **This game tape.** It didn't happen after the last push. Superman pushed four times today and Coach wrote zero game tapes until Steve noticed. The rule existed in config for hours before it was enforced.
- **Root cause:** The `.squad/coach/game-tape/` folder didn't exist in this repo. Superman should have created it when the rule was added, not waited for a complaint.
- **Secondary root cause:** Superman doesn't have a reliable forcing function to actually invoke Coach. The rule says "trigger Coach after every push" but nothing stops Superman from skipping it.

### Pattern to flag
The squad is good at writing rules. Less good at running them the first time. Three examples this session: Coach after push (forgot), game-tape folder (never created), copilot-instructions.md non-profit context (survived two Kryptonite passes before Steve caught it). The squad spots its own gaps only when forced.
