## Required Before Eric Starts

### Vision
- [ ] Right user named — not "users", a specific person or role
- [ ] Right problem defined — what breaks if this isn't built?
- [ ] Right now confirmed — why this task, why this sprint?

### Architecture (Bob)
- [ ] Blueprint written and filed in `.squad/decisions/`
- [ ] Schema / data model defined
- [ ] API contracts agreed
- [ ] Key risks identified and mitigated or accepted

### Design (Marlo)
- [ ] UX spec written — layouts, flows, empty states, error states
- [ ] Edge cases caught in the spec, not left for QA

### Plan (Paulien)
- [ ] Task broken into ordered subtasks
- [ ] Dependencies identified
- [ ] Done-criteria written for each subtask
- [ ] Estimate reviewed — realistic, not optimistic

### Quality (Vera)
- [ ] Acceptance criteria written — specific, testable
- [ ] Test scenarios identified upfront

### Red Team (Kryptonite) — Squad and Full mode only
- [ ] Plan attacked — Kryptonite has reviewed and flagged
- [ ] Critical findings actioned or accepted before build starts

### Economics (G) — Full mode and any external product
- [ ] Economics defined — revenue, cost, or budget line
- [ ] No undefined financial assumptions in the build
